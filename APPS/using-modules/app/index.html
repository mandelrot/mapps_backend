How to use backend modules

Each app has its own backend function modules (internal.js and external.js), and can have any other module to work as we will see in this example. Besides this, the backend has a public folder (/utils/public) where there are some common utilities modules too, with functions that can be called directly from the frontend (from the browser itself) with an specific "utils" socket event.

So there are two ways to use Node Modules from your app. Here will see examples of them:

- First, this webpage will ask directly to the backend to encrypt a text. The backend will look for the generic /utils/public/encryption.js module, use it to cipher the text, and send it back.

- Second, this webpage will ask its own internal.js module to use a self-contained encryption module in the frontend folder. The internal.js module will require its own module, cipher the text, and send it back.

IMPORTANT: the modularity of this system implies that the frontend app modules are totally independent from the backend ones. In this encryption example, please note that the encryption password we use to cipher the texts are also independent and hence different. Using a common encryption password for the frontend apps or an app-specific one for each is your choice.
