How to use backend modules

Each app has its own function modules (two files called "internal.js" and "external.js" located in its own "backend of the app package"), and they can use some basic modules provided by the server or use their own (having them in subfolders and just requiring them).

Besides this, the server has a public folder (/utils/public) where there are some common utilities modules too, with functions that can be called directly from the frontend (from the browser itself) with an specific "utils" socket event.

So there are two ways to use Node Modules from your app. Here will see examples of them:

- First, this webpage will ask directly to the server (the actual backend) to encrypt a text. The backend will look for the generic /utils/public/encryption.js module, use it to cipher the text, and send it back.

- Second, this webpage will ask its own internal.js module to use a self-contained encryption module in the frontend folder. The internal.js module will require its own module, cipher the text, and send it back.

IMPORTANT: the frontend apps may use their own packages as said. But the encryption is a special case, because it needs an "encryption password" and if someone uses the backend encryption module (with its password) and then someone else uses an independent encryption module (with a different password) they both will not be able to work together. The developer must choose between encryption modularity or encryption coherence for their frontend apps.
